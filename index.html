Prérequis :

  • Python 3.x installé sur votre système.
  • Bibliothèque Transformers de Hugging Face installée (pip install transformers).
  • Un modèle pré-entraîné GPT-2 et son tokenizer.
  • FastAPI
  • Uvicorn

Comment lancer l'interface web ?

se déplacer dans le terminal à l'endroit ou se trouve "main.py" ainsi que deux dossiers static et modele

lancer l'interface à l'aide de la commande ci dessous :

python -m uvicorn main:app --reload

  • La difficulté de la quête à choisir entre easy, medium, hard

Objectif

Le monde étant en grand danger suite aux multiples attaques de monstres. Il nous a semblé urgent de nous entraîner à combattre à l'aide d'un générateur de quêtes automatiques. Les objectifs de cet entraînement sont, aux premiers abords, simples. Afin de générer la quête, il faut mentionner :

  • La carte où se déroulera la quête pour pouvoir s'acclimater à divers environnements de combat.
  • La difficulté de la quête.
  • Le ou les monstres à chasser.

Données utilisées

Les données qui ont été utilisées dans ce projet viennent d'un dépôt GitHub libre : https://github.com/CrimsonNynja/monster-hunter-DB/tree/master

Ce sont des données qui proviennent du jeu vidéo Monster Hunter, ces données sont des quêtes de ce même jeu vidéo : un jeu de combat entre chasseur et monstres hyper classes. La particularité des quêtes de Monster Hunter, c'est qu'elles sont simplement construites : un monstre à tuer, le contexte, l'objectif, la carte et la difficulté. Ce qui est parfait pour notre projet. En plus de ça, ce sont des quêtes qui ont un fond d'humour ce qui nous permet de rire malgré le destin du monde qui s'avère bien noir...

Prétraitement

Nous avons de la chance, les données du dépôt GitHub sont sous un format JSON avec plusieurs types de balises : target, objective, map, difficulty. Afin de nous faciliter la tâche, nous avons décidé de créer notre propre format de balise en abandonnant le format JSON. Pour ce faire, nous avons simplement pris le contenu de chaque balise qui nous intéressait pour l'écrire dans un fichier texte balisé à notre manière. Exemple : Map : Desert | Difficulty : easy | Objective : Kill a Rathian | Description ..

Implémentation

Le modèle que nous utiliserons pour ce projet est GPT2. Nous avons utilisé le modèle medium de GPT2, si nous prenions le large, on serait encore en train d'entraîner le modèle :( . Nous avons incorporé au modèle le corpus au format TXT qui a été pré-traité avec notre format de balise. Le modèle apprend par taille de bloc textuel. Nous avons choisi une taille qui permet d'englober la taille d'une quête entièrement -> 75. Nous avons choisi 10 epochs puis un batch size de 10. L'entraînement a duré 30 minutes.

Le code pour la génération de texte a été plus compliqué à produire. Nous avons mis une limite à max_lenght: 75 max de n-grammes générés.

Temperature : Contrôle le niveau de créativité dans la génération de texte. Une température plus élevée génère des sorties plus variées mais moins cohérentes. Top-K Sampling : Limite le choix de tokens aux K tokens les plus probables à chaque étape. Cela favorise la cohérence tout en introduisant une certaine variété. Top-P Sampling (Nucleus Sampling) : Limite la distribution cumulative des probabilités pour le choix des tokens. Cela permet une génération plus dynamique en échantillonnant parmi les tokens les plus probables jusqu'à ce que la somme des probabilités dépasse un seuil. No Repeat N-Grams : Empêche la répétition immédiate de séquences de mots. Cela garantit que la quête générée ne contienne pas de motifs répétitifs indésirables. Ces paramètres peuvent être ajustés en fonction des besoins pour obtenir des résultats de génération souhaités.
